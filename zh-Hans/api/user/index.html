User API

The User API provides endpoints for authenticated users to manage their own resources: messages, API tokens, statistics, attachments, topics, and push clients.

Base URL​

http://<your-host>:9527/api/user

Authentication​

All User API endpoints require a JWT token in the Authorization header:

Authorization: Bearer <jwt-token>

JWTs are obtained through the Auth API endpoints.

  • Expiry: 24 hours (web panel) or 90 days (client tokens)
  • Roles: admin (sees all resources) or user (sees own resources only)
备注

Admin users can access all resources through these endpoints. Regular users can only access their own resources.

Auth​

Login (Client)​

POST /api/auth/login

Authenticate with email/username and password. Returns a JWT token suitable for client applications (Android, Desktop, CLI).

Request Body:

FieldTypeRequiredDescription
emailOrUsernamestringYesEmail or username.
passwordstringYesAccount password.

Response -- 200 OK:

{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "email": "user@example.com",
      "username": "user",
      "role": "user"
    }
  }
}

Register​

POST /api/auth/register

Create a new user account.

Request Body:

FieldTypeRequiredDescription
emailstringYesEmail address (must be unique).
passwordstringYesPassword (minimum 6 characters).

Change Password​

POST /api/auth/change-password

Change the password of the currently authenticated user. Requires JWT authentication.

Request Body:

FieldTypeRequiredDescription
currentPasswordstringYesCurrent password.
newPasswordstringYesNew password (minimum 6 characters).

Messages​

List Messages​

GET /api/user/messages

Retrieve a paginated list of your messages.

Query Parameters:

ParameterTypeDefaultDescription
pagenumber1Page number (1-based).
pageSizenumber50Items per page (max 500).
statusstring--Filter by status: queued, sent, failed, dead.
topicstring--Filter by topic ID.

Response -- 200 OK:

{
  "success": true,
  "data": {
    "items": [
      {
        "id": "msg-uuid-1",
        "channelType": "push",
        "toAddress": "device-uuid",
        "subject": "Hello",
        "body": "World",
        "status": "sent",
        "createdAt": 1719849600
      }
    ],
    "total": 150,
    "page": 1,
    "pageSize": 50
  }
}

Get Message​

GET /api/user/messages/{id}

Retrieve a single message by ID.

Token Management​

API tokens are used by applications to authenticate with the Send API. Each token has scopes, rate limits, and optional IP whitelisting.

List Tokens​

GET /api/user/tokens

Retrieve your API tokens. Admin users see all tokens; regular users see only their own.

Response -- 200 OK:

{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Production App",
      "token": "nh_aBcDe***",
      "scopes": ["email", "sms"],
      "rateLimit": 100,
      "enabled": true,
      "lastUsedAt": 1719849600000,
      "createdAt": 1719800000000
    }
  ]
}

Create Key​

POST /api/user/tokens

Create a new API key. The full key value is returned only on creation.

Request Body:

FieldTypeRequiredDescription
namestringYesHuman-readable name (1-100 characters).
scopesstring[]NoArray of channel types. Default: ["email", "sms", "push"].
rateLimitnumberNoMax requests per minute. Default: 100. Range: 1-10,000.
ipWhiteliststring[]NoArray of allowed IP addresses. Empty means no restriction.

Generate Client Token​

POST /api/user/tokens/generate-client-token

Generate a JWT token for client applications (Android, Desktop). The token has a 90-day expiry.

Get Token​

GET /api/user/tokens/{id}

Retrieve a single token with its full value.

Update Token​

PUT /api/user/tokens/{id}

Update token settings. All fields are optional.

Delete Token​

DELETE /api/user/tokens/{id}

Permanently delete an API token.

Rotate Token​

POST /api/user/tokens/{id}/rotate

Generate a new token value for an existing token. The old value is immediately invalidated.

Topics​

List Topics​

GET /api/user/topics

Retrieve your topics with pagination and search.

Query Parameters:

ParameterTypeDefaultDescription
limitnumber50Items per page (max 200).
offsetnumber0Offset for pagination.
searchstring--Search by name or display name.

Create Topic​

POST /api/user/topics

Create a new topic, optionally forking from an existing one.

Request Body:

FieldTypeRequiredDescription
namestringYesUnique topic name.
displayNamestringNoHuman-readable display name.
descriptionstringNoTopic description.
iconstringNoIcon URL or data URI.
forkFromstringNoSource topic ID to fork from. Copies displayName, description, and icon unless overridden.

Get Topic​

GET /api/user/topics/{id}

Update Topic​

PUT /api/user/topics/{id}

Request Body: All fields optional — name, displayName, description, icon.

Delete Topic​

DELETE /api/user/topics/{id}

Fork Topic​

POST /api/user/topics/{id}/fork

Create a new topic by forking from an existing topic. Copies the source topic's icon. displayName and description can be overridden.

Request Body:

FieldTypeRequiredDescription
namestringYesNew topic name (must be unique).
displayNamestringNoOverride display name.
descriptionstringNoOverride description.

Push Clients​

Register Push Client​

POST /api/user/push/register

Register or update a push client device.

Request Body:

FieldTypeRequiredDescription
uuidstringYesDevice UUID.
namestringNoDevice name.
osstringNoOperating system.
archstringNoCPU architecture.
desktopstringNoDesktop environment.
appVersionstringNoApp version.
fcmTokenstringNoFirebase Cloud Messaging token.

Update Push Client​

PATCH /api/user/push/client

Update push client name or FCM token.

Ack Messages​

POST /api/user/push/ack

Acknowledge delivered push messages so they won't be re-delivered.

Poll Messages​

GET /api/user/push/poll

Long-poll for undelivered push messages.

Stream Messages (SSE)​

GET /api/user/push/stream

Server-Sent Events stream for real-time push delivery. Requires ?uuid=<device-uuid> and either Authorization header or ?token=<jwt>.

WebSocket​

GET /api/user/push/ws

WebSocket connection for real-time push delivery. Requires ?uuid=<device-uuid> and ?token=<jwt>.

List Push Clients​

GET /api/user/push/clients

List your registered push clients.

Delete Push Client​

DELETE /api/user/push/clients/{uuid}

Delete one of your push clients by UUID.

Clients​

List Clients​

GET /api/user/clients

List your registered push clients (same as List Push Clients).

Statistics​

Overview Statistics​

GET /api/user/stats/overview

Get aggregate message statistics. Admin users see instance-wide stats; regular users see their own.

Response -- 200 OK:

{
  "success": true,
  "data": {
    "totalMessages": 15234,
    "sentMessages": 14800,
    "failedMessages": 434,
    "queuedMessages": 12,
    "successRate": 97.15,
    "messagesLast24h": 523,
    "messagesLast7d": 3412
  }
}

Daily Statistics​

GET /api/user/stats/daily

Get per-day message counts for the last 7 days.

Channel Statistics​

GET /api/user/stats/channels

Get message counts grouped by channel type.

Recent Messages​

GET /api/user/stats/recent

Get the 10 most recent messages.

Attachments​

List Attachments​

GET /api/user/attachments

List your uploaded attachments with pagination.

Get Attachment Stats​

GET /api/user/attachments/stats

Get storage usage statistics.

Batch Delete Attachments​

POST /api/user/attachments/batch-delete

Delete multiple attachments by ID.

Delete Attachment​

DELETE /api/user/attachments/{id}

Delete a single attachment.

Download Attachment​

GET /api/user/attachments/{id}/download

Download an attachment file.

User Settings​

Get Settings​

GET /api/user/settings

Get the current user's settings.

Update Settings​

PUT /api/user/settings

Update the current user's settings.
